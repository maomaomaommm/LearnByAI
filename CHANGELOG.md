# 系统版本演进与未来规划总结
*(From MVP to Fullstack Beta)*

这份总结详细盘点了系统从最初的 MVP 版本（仅能简单生成大纲和内容）到当前版本的核心架构演进，以及接下来系统还需要验证、优化和开发的重要模块，非常适合同步给开发团队。

---

## 🚀 第一部分：从 MVP 到当前版本的新增核心特性

### 1. 全栈架构升级与持久化 (Backend & DB)
- **接入 Supabase**：从原本纯前端+临时内存状态，升级为完整的账号鉴权和数据库持久化系统。用户的课程和生成记录现在可以落盘。
- **引入异步任务队列系统**：彻底重构了生成流程。加入了 `GenerationJob` 任务队列机制，将大模型漫长的生成过程完全放入后台剥离，前端通过 API 轮询状态。避免了原来长时间等待导致浏览器请求超时的致命问题。

### 2. “按需生成”懒加载策略 (Lazy Generation)
- MVP 版本是一次性生成大量内容，容易造成大模型拥堵和 Token 浪费。
- **新版策略**：现在只在创建课程时生成全局架构（Course Bible）和目录。随后系统仅自动开始生成**第一章**的全文。后续所有章节均默认挂起（Pending），只有当用户主动点进该章节页面时，才会触发后台生成逻辑。极大地节省了算力并提升了响应体验。

### 3. 高稳定性与系统容错 (Resilience & Stability)
- **超长上下文支持**：考虑到 Kimi 生成两三万字长文需要数分钟，系统将生成请求的底层生命周期（Timeout）限制从 120 秒大幅放宽至 15 分钟。
- **阻断 OOM 内存泄漏**：彻底修复了因并发状态更新导致 React 前端无限轮询 `GET /api/courses`，进而引发 Node.js 后台 CPU 和内存满载崩溃的严重 Bug。
- **格式护栏 (Format Guards)**：增加了模型输出的强格式校验，确保 Markdown 解析不会因为大模型偶尔的幻觉而导致页面渲染白屏。

### 4. 极致 UI 与本土化 (UI/UX & Localization)
- **全新黑客极客风设计**：引入了高对比度暗色主题，以及 `Inter`, `Outfit`, `JetBrains Mono` 字体组合，整体质感大幅提升。
- **深度中文化**：清理了页面上原有的 `Gemini 3.1 Pro` 等写死的英文假数据，并将复杂的“生成状态流”和所有提示错误信息全部完美汉化。

## 🛠 第二部分：未来需要检验、优化与开发的 Roadmap

虽然系统当前可用性已大幅提升，但要达到真正的生产级（Production Ready），团队后续需要重点关注以下三个方向：

### 🔍 1. 需要检验的模块 (To Verify)
- **并发压力测试**：目前的任务队列是跑在单实例 Node.js 中的。如果在多用户场景下同时发起了十几个 `AUTHOR` 长文本生成任务，内存是否会再次吃紧？需要验证在服务器上的真实并发承载力。
- **数据库竞态条件检验**：后台在同时完成多个章节生成时，并发写入本地 JSON（`local-beta-store.json`）和同步到云端 Supabase 时是否存在死锁或内容被覆盖的隐患。
- **长文本质量衰减测试**：虽然模型能生成很长的文本，但在“按需生成”到最后几章时，模型是否依然能精准记住第一章的全局设定（Course Bible），需要人工抽检。

### ⚡ 2. 需要优化的模块 (To Optimize)
- **从 HTTP 轮询升级为 SSE/WebSocket**：目前前端依然是依靠每 2.5 秒发送一次 HTTP 请求来拉取进度。后续强烈建议升级为 Server-Sent Events (SSE) 或 WebSocket 长链接，由后端主动推送状态，降低服务器请求压力。
- **流式字字渲染 (Streaming UI)**：目前用户在等待章节生成时，只能看着 "GENERATING..." 转圈，直到全篇生成完毕才一次性展示。为了降低用户的焦虑感，需要将底层的流式 Chunk 数据直接透传到前端实现“打字机”式的实时渲染。

### 🏗 3. 还需要开发的全新功能 (To Develop)
- **TUTOR 伴读问答系统**：当前右侧的 `TUTOR TERMINAL` 面板还是一个空壳。需要开发“用户双击正文段落即可划线，并在右侧唤起大模型进行针对性答疑”的交互功能，这是项目的核心护城河。
- **导出功能 (Export)**：完成 PDF / ePub 格式的电子书一键导出功能（目前后端有存根但未完全实装）。
- **课程管理仪表盘 (Dashboard)**：用户需要一个个人中心来查看自己曾经创建过的所有课程、管理学习进度或删除不需要的课程。
- **移动端适配优化 (Mobile Responsive)**：目前复杂的左右双栏结构在手机端可能体验不佳，需要开发折叠侧边栏逻辑以适配移动端阅读。
