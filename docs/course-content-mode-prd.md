# LearnByAI 讲义模式 / 教材模式 PRD

状态：讨论稿  
日期：2026-07-09  
适用范围：课程创建流程、课程列表展示、课程生成策略、未来教材专用排版

## 1. 背景

当前 LearnByAI 生成的课程内容更接近“讲义”：结构清楚、生成速度相对快、适合用户快速了解某个领域、建立知识地图和学习路线。

但如果用户希望长期系统研读，当前内容会显得不够“教材化”：动机铺垫、贯穿案例、推导细节、章节衔接、小结、习题和教材式插图都还不够完整。

因此需要在产品上明确区分两种课程形态：

- 讲义模式：快速学习、快速了解、快速建立体系。
- 教材模式：深度学习、系统研读、内容更完整，可配套专门的教材排版。

## 2. 产品目标

1. 保留当前 LearnByAI 的快速生成优势，将现有生成形态沉淀为“讲义模式”。
2. 新增“教材模式”，面向更详细、更系统、更接近正式教材的生成结果。
3. 在创建课程前让用户明确选择内容模式，避免用户对生成耗时、内容密度和成本预期不一致。
4. 在“我的课程”列表中支持按讲义 / 教材筛选，并让两类课程在外观上有明显区分。
5. 为后续“教材专用阅读排版 / 导出排版 / 插图策略”预留产品和技术空间。

## 3. 模式定义

### 3.1 讲义模式

定位：快速学习某个领域，建立核心知识地图。

适合场景：

- 用户想快速了解一个新领域。
- 用户希望生成一套学习提纲、课程讲义或复习材料。
- 用户更关注速度、覆盖面和重点梳理。

内容特征：

- 章节相对短。
- 概念讲解偏直接。
- 重点、框架、步骤、结论更突出。
- 示例和推导适量，不追求完整教材式展开。
- 当前 LearnByAI 的生成效果可作为讲义模式的基线。

成本与体验预期：

- 生成速度更快。
- Token 消耗更低。
- 更适合高频创建和快速试错。

### 3.2 教材模式

定位：生成可长期阅读、系统研读、接近正式教材的深度内容。

适合场景：

- 用户想系统掌握一个领域。
- 用户希望内容像一本小型教材，而不是课堂讲义。
- 用户愿意等待更长生成时间，并接受更高 token 消耗。

内容特征：

- 章节更长，内容更完整。
- 每章需要更强的教材结构，例如：动机、主例、定义、推导、解释、反例 / 误区、应用、小结和练习。
- 章节之间需要更强的衔接：上一章遗留的问题、本章解决的问题、下一章的自然引出。
- 可以引入贯穿案例，例如强化学习课程中反复使用 gridworld、多臂老虎机、库存控制等主例。
- 插图更偏教材 / 学术图，而不是产品介绍图或装饰图。

成本与体验预期：

- 生成时间更长。
- Token 消耗更高。
- 更适合低频、慎重创建、长期阅读和导出。

## 4. 创建课程流程改动

### 4.1 当前流程

在“我的课程”页面点击“创建课程”后，当前会直接进入课程创建表单页面，也就是截图一中的“定制你的专属课程”界面。

当前路径认知：

- 用户点击“创建课程”。
- 直接进入课程创建表单。
- 用户填写主题、目标、基础、讲解风格等信息。
- 提交后开始生成课程。

### 4.2 新流程

点击“创建课程”后，不再直接进入课程创建表单，而是先进入新增的“选择课程模式”界面。

新流程：

1. 用户在导航栏或“我的课程”页面点击“创建课程”。
2. 系统进入新增页面：选择课程模式。
3. 页面左右展示两种模式：
   - 左侧：讲义模式。
   - 右侧：教材模式。
4. 用户选择其中一种模式。
5. 系统再跳转到当前的课程创建表单页面。
6. 后续创建课程时，该课程携带用户选择的内容模式。

建议路由，仅作为讨论稿：

- 新增模式选择页：`/create/mode` 或 `/create/select-mode`
- 现有创建表单：继续使用 `/create`
- 跳转时携带参数：`/create?mode=lecture` 或 `/create?mode=textbook`

### 4.3 模式选择页要求

页面结构：

- 保持 LearnByAI 当前简洁、克制的视觉风格。
- 主体区域只聚焦一件事：选择课程生成模式。
- 左右两张模式卡片，分别代表讲义模式和教材模式。
- 两种模式需要有清晰文案差异，而不是“低配 / 高配”的暗示。

讲义模式卡片建议表达：

- 标题：讲义模式
- 副标题：快速生成，适合快速理解一个领域
- 要点：
  - 生成更快
  - 内容更精炼
  - 适合入门、复习、建立知识地图

教材模式卡片建议表达：

- 标题：教材模式
- 副标题：深度生成，适合系统研读
- 要点：
  - 内容更完整
  - 推导、例子、练习更充分
  - 生成更慢，消耗更多 token

交互要求：

- 点击任一卡片即可选择并继续。
- 可以有一个默认推荐，例如默认推荐“讲义模式”，但不要让“教材模式”显得像隐藏高级功能。
- 教材模式需要明确提示“生成时间更长、消耗更多 token”。

## 5. “我的课程”列表改动

### 5.1 当前状态

截图二中，“我的课程”页面以卡片列表展示用户创建的课程。当前所有课程外观基本一致，没有讲义 / 教材的区分，也没有模式筛选。

### 5.2 新增筛选

“我的课程”页面需要支持按内容模式筛选。

筛选项：

- 全部
- 讲义
- 教材

建议表现：

- 使用 segmented control 或紧凑 tab，不建议做成复杂筛选器。
- 默认显示“全部”。
- 用户选择“讲义”时只展示讲义模式课程。
- 用户选择“教材”时只展示教材模式课程。

### 5.3 课程卡片外观区分

讲义和教材需要在卡片上有明显但克制的视觉区分。

可选方案：

- 在课程卡片标题附近增加模式 badge：`讲义` / `教材`。
- 讲义卡片使用更轻量的标识，例如细线 badge。
- 教材卡片使用更稳定、更“书籍感”的标识，例如深色小标签或书本图标。
- 教材卡片可以在元信息中展示“教材模式”以及更完整的章节 / 生成状态。

设计原则：

- 不能让页面变花。
- 不能让教材卡片像营销卡片。
- 区分要足够明显，用户一眼能知道这门课是讲义还是教材。
- 和当前 LearnByAI 的白底、细线、克制排版保持一致。

## 6. 教材专用排版方向

教材模式后续需要考虑专门的阅读和导出排版，而不只是沿用当前网页讲义样式。

方向：

- 标题层级更像书，而不是网页内容块。
- 定义、定理、例子、证明、提示、误区、练习等模块有统一样式。
- 图注统一，例如“图 2-1”。
- 章节末尾固定包含：本章小结、关键术语、概念检查、练习题、延伸阅读。
- PDF / 打印样式应更接近教材版式。

本阶段只记录方向，不立即实现。

## 6.1 教材模式左侧目录

教材模式的阅读器左侧目录需要和讲义模式明显区分。

当前讲义模式的左侧目录更接近章节导航：只展示课程目录中的章节，点击章节后进入对应章节内容。

教材模式的左侧目录应更接近正式教材、论文或书籍目录，参考用户提供的图二样式：

- 左侧目录展示全书结构，而不只是章节列表。
- 目录中需要体现章节与小节的层级关系。
- 章节使用类似“第一章 / 第二章”或“01. / 02.”的编号体系。
- 小节使用类似“1.1 / 1.2 / 2.1”的编号体系。
- 小节标题缩进展示，形成清晰的层级感。
- 可以保留当前 LearnByAI 的克制、轻量视觉风格，不需要完全复刻纸质目录中的页码和点线。

小节展示要求：

- 小节标题需要支持展开 / 收起。
- 用户可以选择展示小节，也可以选择隐藏小节。
- 至少需要支持按章节展开 / 收起小节。
- 可以进一步考虑提供一个全局开关：显示全部小节 / 隐藏全部小节。
- 引言章和总结章如果没有小节，则只展示章节项。

跳转要求：

- 点击章节标题：跳转到对应章节页面，行为和当前讲义模式一致。
- 点击小节标题：跳转到该章节内对应小节位置。
- 如果用户点击的是当前章节内的小节，则在当前页面内滚动到对应小节锚点。
- 如果用户点击的是其他章节下的小节，则先跳转到该章节页面，再定位到对应小节锚点。
- 小节锚点需要稳定，不能因为标题轻微变化导致历史链接失效。

右侧正文展示要求：

- 网页端右侧章节内容仍然按“一章一页 / 一章一章跳转”的方式展示，保持和当前模式一致。
- 不做整本教材一次性长页面加载。
- 小节跳转只是章节页内部的定位能力，不改变章节级阅读模型。

交互与状态要求：

- 左侧目录需要高亮当前章节。
- 当用户滚动到某个小节时，可以高亮当前小节，作为增强项。
- 展开 / 收起状态可以先做页面内状态，后续再考虑持久化到用户偏好。
- 移动端需要考虑目录抽屉或折叠菜单，避免占用正文阅读空间。

数据要求：

- 教材模式需要保存章节与小节的结构化大纲。
- 每个小节需要有稳定的 `sectionId` 或 slug。
- 正文渲染时，小节标题需要带对应锚点。
- 左侧目录应从结构化大纲生成，而不是临时从正文 Markdown 中脆弱解析。

## 6.2 TeX / PDF 教材排版与导出约定

教材模式需要有正式的 TeX 渲染和 PDF 导出约定。网页阅读器解决在线阅读体验，TeX / PDF 导出解决“像一本教材”的最终出版物形态。

参考《Mathematical Foundations of Reinforcement Learning》PDF 后，记录以下可借鉴方向：

- 目录页展示 chapter / section / subsection，使用页码和点线引导。
- 正文页有运行页眉，页眉展示当前节标题，底部居中页码。
- 章首页使用大号“Chapter N”和章标题，并在章首放置全书地图 / 章节定位地图。
- 图、表、算法、灰盒、定理、公式都采用全书统一的编号和引用方式。
- 公式居中展示，编号右对齐，例如 `(2.11)`。
- 图题、表题、算法题、盒子标题都使用统一格式，例如 `Figure 2.1`、`Table 4.1`、`Algorithm 4.1`、`Box 2.1`。
- 灰盒用于证明、补充材料或可选阅读内容，标题栏清晰，正文背景浅灰。
- 章节之间通过 Overview / map / chapter intro 强化教材主线。

### 6.2.1 导出目标

教材模式的 PDF 导出应优先走 TeX 编排，而不是简单的浏览器打印。

建议技术方向：

- 中文教材使用 `XeLaTeX` 或 `LuaLaTeX`。
- TeX 文档类建议使用 `ctexbook`，以支持中文章节标题、目录和中文字体。
- 数学公式使用 `amsmath`、`amssymb`、`mathtools` 等标准数学包。
- 交叉引用使用 `hyperref` + `cleveref` 或类似机制。
- 表格使用 `booktabs` / `longtable`，不要把表格截图化。
- 算法环境使用统一的 algorithm 环境或自定义 `tcolorbox` 风格。
- 灰盒、定义、例子、提示等块使用统一的 theorem-like / `tcolorbox` 环境。

讲义模式可以继续使用当前较轻量的导出方式；教材模式需要单独的 TeX 导出模板。

### 6.2.2 教材 PDF 文档结构

建议导出结构：

1. 封面 / 标题页。
2. 元信息页，可包含主题、生成时间、用户目标、模型信息等。
3. 目录。
4. 可选前言。
5. 教材地图 / 全书 Overview。
6. 正文 main matter：
   - 第一章固定为“引言”。
   - 中间章节为教材主体。
   - 最后一章固定为“总结”。
7. 可选附录。
8. 参考文献。
9. 符号表 / 术语表。
10. 可选索引。

页码约定：

- 前置部分可使用罗马页码。
- 正文部分使用阿拉伯数字页码。
- PDF 书签需要和章节 / 小节结构一致。

### 6.2.2.1 封面 / 标题页约定

教材模式需要有专门的封面 / 标题页模板，不能直接使用网页标题或课程卡片样式。

这里的“封面”优先指 TeX / PDF 导出的第一页标题页，类似当前 PDF 第一页的白底居中封面，而不是营销海报、宣传图或大面积插画封面。

封面目标：

- 一眼看起来像一本正式教材，而不是课程讲义或网页导出。
- 和 LearnByAI 网页风格保持关系：克制、干净、低饱和、留白充足。
- 不使用营销海报式封面，不使用复杂渐变、3D、装饰性背景或大面积插画。
- 封面应适合 PDF 阅读、打印和分享。

封面必备信息：

- 教材标题。
- 副标题或一句简短说明，可选。
- 生成主体：`LearnByAI`。
- 生成日期。
- 课程模式：`教材模式`。
- 可选：学习目标摘要。
- 可选：读者基础 / 难度。
- 可选：作者信息，例如用户昵称或“为某用户生成”。

封面视觉元素：

- 可以使用教材地图的简化版作为封面视觉元素。
- 可以使用一张模型生成的抽象学术插图，但必须服务主题，不做纯装饰。
- 如果使用插图，插图应低饱和、白底或浅色背景，避免抢正文标题。
- 封面不强制每本教材都生成独立大图；早期可以先使用统一模板 + 主题色 + 简化地图。

版式建议：

- 标题居中或左对齐均可，但全站需要统一。
- 默认优先采用白底、大留白、中央标题块的标题页样式，与当前 PDF 封面保持连续性。
- 标题层级清晰，主标题最大，副标题次之，元信息最弱。
- 主标题应使用系统归纳后的正式教材名，不直接把用户创建课程时输入的长 prompt 放到封面标题。
- 用户输入中的参考书、学习目标、特殊要求等信息应放在副标题、元信息页或前言中，而不是全部挤在封面。
- 封面底部放生成日期、LearnByAI 标识和可选说明。
- 不在封面放过多正文摘要。

TeX 实现建议：

- 封面使用单独的 `titlepage` 环境。
- 封面不参与正文页码编号。
- 封面之后可接元信息页，再进入目录。
- 封面所需字段由教材元数据提供，不从正文 Markdown 中解析。

待讨论：

- 封面是否允许用户选择风格。
- 封面是否默认生成主题插图。
- 封面是否展示用户姓名 / 昵称。
- 封面是否展示模型和生成时间等技术信息，还是放到元信息页。

### 6.2.3 编号体系

编号必须由 TeX 自动生成，不能由模型在正文里手写死编号。

章节编号：

- 章：`第 1 章` / `Chapter 1`
- 节：`1.1`
- 小节：`1.1.1`
- 默认不建议超过三级标题；更深层级可用列表或段落标题。

内容块编号暂定按章编号，章内连续，跨章通过章号保证全书唯一：

- 定义：`定义 2.1`、`定义 2.2`
- 例子：`例 2.1`、`例 2.2`
- 定理：`定理 2.1`
- 命题：`命题 2.1`
- 引理：`引理 2.1`
- 推论：`推论 2.1`
- 算法：`算法 2.1`
- 图：`图 2.1`
- 表：`表 2.1`
- 盒子 / 补充材料：`框 2.1` 或 `Box 2.1`
- 公式：`(2.1)`、`(2.2)`

“前后延续”的含义：

- 同一类型的编号在同一章内连续递增。
- 不允许同一章内出现两个 `图 2.1` 或两个 `定义 2.1`。
- 跨章节引用必须带完整编号，例如“见图 3.2”“由式 (4.5) 可得”。
- 模型只能产生语义块和 label，最终编号由 TeX counter 决定。

如后续产品上希望采用“全书连续编号”而不是“按章编号”，可以切换 counter 策略；当前建议优先按章编号，因为教材读者定位更自然。

### 6.2.4 公式约定

公式需要全教材统一。

基本约定：

- 所有数学公式都以 LaTeX 源码形式存储和渲染，不使用公式截图。
- 行内公式使用统一的 inline math。
- 重要展示公式使用 `equation`、`align`、`gather`、`multline` 等标准环境。
- 被正文引用的公式必须编号。
- 临时推导公式可以不编号，但样式必须和编号公式一致。
- 公式编号格式统一为 `(章号.公式序号)`。
- 公式引用统一使用 `\eqref{...}` 或等价机制，正文显示为“式 (2.3)”。

符号一致性：

- 教材模式需要维护全书符号表。
- 同一个概念全书使用同一符号，例如状态集合、动作集合、策略、价值函数、折扣因子等。
- 符号首次出现时需要定义，后续不得随意更换。
- 复杂符号建议通过 TeX macro 统一，例如 `\newcommand{\Vpi}{V^\pi}` 这类宏可由导出层维护。

生成约束：

- 模型不得混用 Unicode 近似符号、Markdown 数学、HTML 公式和 TeX 公式。
- 不允许出现无法编译的裸 `$`、未闭合环境或缺失大括号。
- 导出前需要做 TeX 编译校验。

### 6.2.5 定义、例子、证明与灰盒

教材模式需要把定义、例子、定理、证明和补充材料做成稳定环境。

建议环境：

- `definition`：用于正式概念定义。
- `example`：用于示例，尤其是贯穿案例。
- `theorem` / `proposition` / `lemma` / `corollary`：用于数学结论。
- `proof`：用于证明。
- `remark`：用于备注、直觉解释或常见误区。
- `box`：用于可选阅读、证明细节、补充推导、扩展材料。

样式方向：

- 定义和例子要明显区别于普通正文，但不要过度花哨。
- 证明或补充材料可以使用浅灰背景盒，参考示例 PDF 中 `Box 2.1` 的做法。
- 定理正文可以采用“编号 + 标题 + 正文”的紧凑格式。
- 所有环境都必须支持 label 和 cross-reference。

### 6.2.6 图、表、算法

图：

- 图编号按章连续，例如 `图 4.2`。
- 图题统一放在图下方。
- 教材地图、章节定位图、概念图、流程图都纳入统一 figure counter。
- 模型生成图片需要保存高分辨率版本，供 PDF 导出使用。
- 图像在 TeX 中通过 `figure` 环境插入，不能只作为 Markdown 图片散落在正文。

表：

- 表编号按章连续，例如 `表 4.1`。
- 表题统一，建议放在表下方以贴近参考 PDF；若后续设计选择中文教材常见的表题上置，需要全书统一。
- 表格内容优先使用 TeX 表格环境，不使用截图。
- 大表使用 `longtable` 支持跨页。

算法：

- 算法编号按章连续，例如 `算法 4.1`。
- 算法使用统一盒子样式，标题栏清晰。
- 算法步骤需要支持缩进、初始化、目标、循环、条件等结构。
- 伪代码不能作为普通段落混排。

### 6.2.7 引用与 label 规范

所有可引用对象都需要稳定 label：

- 章节：`ch:introduction`
- 小节：`sec:bellman-equation`
- 定义：`def:state-value`
- 例子：`ex:gridworld-return`
- 公式：`eq:bellman-equation`
- 图：`fig:book-map`
- 表：`tab:transition`
- 算法：`alg:value-iteration`
- 盒子：`box:convergence-proof`

约定：

- 正文中不得硬写“图 2.1”这种最终编号，必须写成引用目标。
- 导出层负责把引用渲染成最终编号。
- 如果章节重排或插入新图表，编号应自动更新。
- TeX 编译后不得出现 `??`、`undefined reference` 或断裂引用。

### 6.2.8 TeX 导出校验

教材模式 PDF 导出必须做自动校验。

最低校验：

- TeX 至少编译两轮，确保目录和交叉引用稳定。
- 检查 `.log` 中是否存在 undefined references。
- 检查是否存在未解析引用 `??`。
- 检查公式编号、图编号、表编号是否重复。
- 检查目录、书签、页码是否存在。
- 检查图片文件是否缺失。

视觉校验：

- 渲染 PDF 前几页、章首页、普通正文页、图表页、公式密集页、算法页、灰盒页。
- 检查是否有文本溢出、图表越界、公式截断、表格过宽、页眉页脚错误。
- 教材模式导出在进入正式可用前，需要建立样张测试集。

### 6.2.9 数据结构要求

为了支持 TeX/PDF 导出，教材内容不能只是普通 Markdown 字符串。

需要逐步引入结构化块：

- 段落 block。
- 标题 block。
- 公式 block。
- 定义 block。
- 例子 block。
- 图 block。
- 表 block。
- 算法 block。
- 证明 / 灰盒 block。

每个结构化块至少需要：

- `id`
- `type`
- `label`
- `title`
- `content`
- `chapterId`
- `sectionId`
- `numberingScope`
- `references`

这样才能保证网页阅读、TeX 导出、PDF 导出和后续局部修订共享同一套权威结构。

## 7. 数据与系统影响

需要为课程增加内容模式字段。

建议字段：

- `contentMode: "lecture" | "textbook"`

需要影响的模块：

- 创建课程表单提交。
- 课程存储 payload / 数据库字段。
- 课程列表接口和展示。
- 课程生成 prompt / agent 策略。
- 课程详情页和章节页展示。
- 导出 / 打印样式。
- 后续插图生成策略。

兼容策略：

- 历史课程没有 `contentMode` 时，默认视为 `lecture`。
- 当前生成逻辑默认归入讲义模式。

## 8. 生成策略差异

### 8.1 讲义模式生成策略

讲义模式基本沿用现有生成策略。

重点：

- 快速生成。
- 章节结构清晰。
- 内容覆盖核心概念。
- 不强制完整教材结构。

### 8.2 教材模式生成策略

教材模式需要独立的生成要求。

教材模式不是简单把讲义模式写得更长，而是采用分层规划与分层生成：

- 先生成整本教材的大纲。
- 再生成每个章节的大纲。
- 再生成每个小节的大纲。
- 最后按小节并行生成正文，并用上层大纲约束前后延续性。

每章建议包含：

- 学习动机。
- 贯穿案例或主例。
- 核心定义。
- 直觉解释。
- 必要推导。
- 反例 / 常见误区。
- 方法或算法步骤。
- 应用场景。
- 本章小结。
- 练习题。

生成成本提示：

- 教材模式生成时间会更长。
- 教材模式 token 消耗更高。
- 产品界面需要在用户选择前讲清楚。

## 8.3 教材模式章节结构约束

教材模式的首尾章节固定：

- 第一章固定为“引言”。
- 最后一章固定为“总结”。

### 引言章

引言章可以不拆分小节。

引言章需要说明：

- 学科或主题的背景。
- 为什么这个主题值得系统学习。
- 该领域的发展脉络。
- 当前研究近况。
- 当前前沿信息和开放问题。
- 本教材的学习路线、读者预期和阅读方式。

### 总结章

总结章可以不拆分小节。

总结章需要包含：

- 对全书核心内容的回顾。
- 各章节之间主线的再次串联。
- 对后续学习路径的建议。
- 对领域未来发展的展望。
- 对读者的鼓励、肯定与赞赏。

总结章的语气应当温和、有收束感，避免空泛鸡汤，但可以明确承认读者完成学习所付出的努力。

## 8.4 教材模式分层大纲机制

教材模式需要维护一套类似“教材工程文件”的 Markdown 大纲体系，用于约束全书一致性、章节延续性和并行生成边界。

### 全书大纲 Markdown

全书需要有一个总大纲 Markdown。

用途：

- 描述整本教材的主题、读者画像、难度、教学目标。
- 列出所有章节。
- 明确章节之间的前后关系。
- 说明每章解决什么问题，以及如何自然引出下一章。
- 记录贯穿案例、术语约定、符号约定和整体叙事主线。

全书大纲需要先于所有章节正文生成。

### 章节大纲 Markdown

每个章节需要有一个章节大纲 Markdown。

用途：

- 明确本章在全书中的位置。
- 说明本章承接上一章的什么问题。
- 说明本章要解决的核心问题。
- 列出本章小节。
- 定义本章核心概念、例子、图表需求、练习方向。
- 说明本章结尾如何引出下一章。

章节大纲应当由全书大纲约束生成，不能脱离全书主线。

### 小节大纲 Markdown

每个小节需要有一个小节大纲 Markdown。

用途：

- 明确小节目标。
- 说明小节输入上下文：来自上一小节或上一章的概念。
- 说明小节输出成果：读者读完后应该理解什么。
- 约束小节中的定义、推导、例子、反例、插图或练习。
- 为小节正文生成提供清晰边界，支持小节级并行生成。

小节大纲应当由章节大纲约束生成。

### 分层生成顺序

建议生成顺序：

1. 生成全书大纲 Markdown。
2. 基于全书大纲生成所有章节大纲 Markdown。
3. 基于每个章节大纲生成该章下所有小节大纲 Markdown。
4. 基于全书大纲、章节大纲、小节大纲生成小节正文。
5. 对同一章节内的小节做章节级整合与过渡修订。
6. 对全书做全局术语、风格、前后引用和章节衔接检查。

这个结构的目的：

- 用全书大纲保证整本教材的统一主线。
- 用章节大纲保证章节之间有延续性。
- 用小节大纲明确并行生成任务边界。
- 允许多个小节并行生成，同时降低内容跑偏和重复解释的风险。

## 8.5 教材模式并行生成原则

教材模式可以并行生成小节，但不能直接无约束并行。

并行前必须具备：

- 全书大纲已确定。
- 对应章节大纲已确定。
- 小节大纲已确定。
- 术语、符号和贯穿案例已有统一约定。

并行生成后的整合步骤：

- 检查相邻小节之间是否有自然过渡。
- 检查同一概念是否重复定义或前后定义不一致。
- 检查例子是否服务同一章的主线。
- 检查小节结尾是否能引向下一小节。
- 检查章节结尾是否能引向下一章。

## 9. 插图策略关联

讲义模式和教材模式的插图策略也可以不同。

讲义模式插图：

- 更偏概念辅助图、流程图、课堂板书式草图。
- 数量可以少。
- 主要帮助用户快速抓住结构。

教材模式插图：

- 更偏教材 / 学术插图。
- 需要贴合章节正文中的具体概念、推导、算法或案例。
- 风格统一、图注规范、可用于阅读和导出。

### 9.1 当前暂定方案：模型原生生图优先

当前阶段暂定：流程图、概念图、教材插图的最终图像主链路转向模型原生生图，不再优先使用代码渲染图作为最终输出。

产品判断：

- 代码渲染的流程图语义更稳定，但整体观感容易偏“工程图 / 临时图”，和教材模式希望达到的出版级插图质感有差距。
- 模型原生生图在整体构图、字体布局、留白、配色和教材感上更自然，更符合教材模式的视觉目标。
- 因此，后续插图策略先以 `image2` 这类图像模型生成为主，而不是继续扩大 Mermaid / SVG / canvas / 程序化流程图渲染的最终图链路。

实施原则：

- 模型直接生成最终插图，优先追求原生构图和整体美感。
- 对流程图、状态图、算法图等结构化图，生成提示词中需要明确给出节点、关系、方向、标签和禁止重复标签等约束。
- 可以把结构化图形说明、伪 JSON diagram spec、章节上下文作为辅助提示词，帮助模型理解图的语义。
- 暂不把代码渲染图作为默认最终图，也不把“程序化文字回填”作为主方案。
- 如果模型原生图出现标签错误、重复标签、错别字或结构错误，优先通过重新生成、简化图中文字、减少标签数量、改写提示词解决。
- 在没有 OCR 自动校验能力的阶段，关键插图需要保留人工预览和手动再生成入口。

风险：

- 模型原生图无法保证所有标签、坐标、公式和箭头关系 100% 正确。
- 图中文字越多，错误风险越高。
- 对严格公式推导、复杂状态机、带大量精确标签的图，需要在生成策略中额外谨慎，必要时可降级为人工确认或后续专门处理。

暂定结论：

- 教材模式插图主方向：全面转向模型生图。
- 代码渲染不再作为默认最终输出方案。
- 代码 / 结构化 spec 可以作为辅助信息存在，但最终图优先由模型生成。

### 9.2 教材地图作为标配

教材模式需要把“教材地图”做成标配能力。

教材地图不是普通装饰插图，而是帮助读者理解整本教材结构、章节依赖关系和当前学习位置的导航型插图。参考用户提供的两张图：

- 全书地图：展示整本书的章节结构、知识模块和章节之间的前后关系。
- 章节定位地图：在每一章开头展示同一张或同一套地图，并高亮当前章节，让读者知道“我现在在这本书的哪里”。

生成原则：

- 教材地图由模型生成，纳入模型原生生图主链路。
- 地图内容来自全书大纲 Markdown、章节大纲 Markdown 和章节依赖关系。
- 生成前应先形成结构化 map spec，用于描述章节节点、模块分组、前置依赖、学习路径和当前章节高亮。
- map spec 可作为提示词输入给图像模型，但最终地图图像由模型生成。
- 不使用代码渲染作为默认最终地图输出。

全书地图要求：

- 在教材模式中默认生成一张全书地图。
- 建议放在引言章中，位于学习路线或全书结构说明附近。
- 展示所有主要章节。
- 展示章节所属模块，例如基础概念、核心方法、进阶方法、应用专题等。
- 展示关键依赖关系和学习路径，不需要展示所有细节边。
- 视觉上应像教材中的“Overview of this book”，但风格要适配 LearnByAI 网页，而不是复刻参考图。

章节定位地图要求：

- 每个正文章开头可以生成一张章节定位地图。
- 章节定位地图应复用全书地图的整体结构，突出当前章节。
- 当前章节需要有明显但克制的高亮。
- 非当前章节弱化显示。
- 引言章和总结章可按需要使用总览地图或简化地图。

视觉风格方向：

- 白底、克制、教材感强。
- 使用 LearnByAI 当前网页的简洁风格：细线、低饱和、充足留白。
- 允许使用一到两个强调色，例如暖橙、深灰或品牌相关强调色。
- 避免产品宣传图、营销图、复杂渐变、3D、玻璃拟态和装饰性背景。
- 图中文字尽量短，节点标题应高度概括，避免长句。
- 地图应适合网页阅读，也适合后续 PDF / 打印导出。

产品价值：

- 让教材模式和讲义模式形成明显区分。
- 让读者在长教材中保持方向感。
- 帮助读者理解章节之间不是孤立内容，而是一条有组织的学习路径。
- 在每章开头提供“当前位置”提示，增强正式教材感。

风险与约束：

- 地图包含较多文字，模型生成可能出现标签不准确、重复或排版拥挤。
- 地图节点文案需要尽量短，优先使用章节编号和短标题。
- 早期可先生成全书地图，再逐步扩展到每章定位地图。
- 没有 OCR 自动校验时，需要保留人工预览和重新生成入口。

本阶段只记录策略方向，是否自动生成插图、生成多少张、何时生成，后续继续讨论。

## 10. 非目标

本 PRD 当前不决定以下问题：

- 教材模式具体 token 定价或额度消耗规则。
- 教材模式是否对所有用户开放。
- 教材模式是否默认生成插图。
- 教材模式是否需要单独导出模板。
- 讲义和教材是否允许互相转换。

## 12. 初步实施阶段

### Phase 1：模式概念入库与 UI 入口

- 新增课程内容模式字段。
- 新增模式选择页。

- 教材模式支持按章节内容生成学术 / 教材风格插图。

## 13. 成功标准

- 用户能在创建课程前明确选择讲义或教材。
